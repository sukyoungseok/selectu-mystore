## 주요 논의

### 1. 프로젝트 목적 재확인

마이스토어는 여행 크리에이터 전용 바이오링크 서비스로, 크리에이터가 여행 콘텐츠 기반으로 MRT 상품을 큐레이션하고 팔로워의 실제 예약을 통해 어필리에이트 수익을 창출하는 구조임. 현재 단계는 개발보다 가설 검증이 우선임을 재확인.

### 2. 자동DM 방식의 우위

바이오 링크 방식은 팔로워가 능동적으로 프로필에 방문해야 하는 수동 구조인 반면, 자동DM은 구매 의도가 있는 팔로워에게만 도달 가능함. DM 오픈율이 현저히 높아 전환 효율이 좋음. DM 툴(ManyChat 등)은 크리에이터가 직접 운영하도록 위임하기로 결정.

### 3. 크리에이터 섭외 전략

인스타그래머는 수경님이 직접 DM 탭핑 + 본인 인스타 서비스 소개 게시로 지인 크리에이터 자연 유입을 유도함. 유튜버는 이메일 콜드아웃리치 방식으로 진행하며, "이미 만들어봤어요" + 클로즈드 테스트 초대 앵글을 핵심으로 함.

### 4. MRT API 개발 범위 판단

MRT Open API(항공/숙소/TNA) 및 MyLink API 구조를 검토함. 크리에이터별 partner_id 관리, mylink_id 발급, 서버사이드 인증 처리 등 MVP 범위를 초과하는 개발 요소가 다수 확인됨. 가설 검증 전 개발 스케일이 목적에서 벗어난다고 판단하여 API 연동 전면 포기 결정. MVP 추적은 utm_content 파라미터 수동 부착으로 대체.

### 5. TripSignal CORS 이슈 대응

배포된 Vercel 사이트에서 TripSignal API CORS 오류 발생. Vercel 서버리스 함수 프록시(`api/prices/[city].js`)를 생성하여 임시 대응 완료. 근본 해결은 TripSignal 개발자(태욱님)에게 CORS 헤더 추가 요청 예정.

### 6. TripSignal CTA 버튼 제거

"더 많은 최저가 경비 찾기" 버튼을 제거하여 모든 유입이 MRT 상품으로 집중되도록 개선함. TripSignal 파인프린트 링크도 텍스트로 변경.

### 7. 첫 타겟 크리에이터: 수코(sookoh)

유튜브 구독자 20.5만명, 나혼자 여행 콘텐츠 특화. 오사카·교토·대만·홍콩 등 MRT 주요 도시와 콘텐츠 겹침이 높음. 코스/경비 정보가 영상에 상세히 포함되어 목업 제작 즉시 가능 판단.
