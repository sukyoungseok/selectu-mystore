# 마이스토어 MVP 전략 확정 — API 포기 후 수동 목업+자동DM 가설 검증 방식

- **날짜:** 2026-04-20
- **주제:** 마이스토어 MVP 전략 재설정 및 크리에이터 섭외 방향 확정
- **유형:** 제품 전략 + 실행 계획
- **배경:** 쿠폰팩 리디자인 배포 후 가설 검증 방법론과 크리에이터 섭외 전략 논의. API 개발 범위가 지나치게 커지는 문제를 인식하고 MVP 범위를 과감히 축소함.

## 최종 결정 사항

**1.** API 연동 개발 전면 포기 — 수동 목업 제작 + 제안 방식으로 가설 검증 집중

**2.** 가설 검증 방식: 크리에이터 자동DM 활용 → 마이스토어 링크 배포 → 클릭·구매 전환 확인

**3.** 크리에이터 섭외 채널 2개: 인스타그래머(직접 탭핑+본인 인스타 게시) / 유튜버(이메일 콜드아웃리치)

**4.** 어필리에이트 추적 방식(MVP): utm_content 파라미터 수동 부착 → 수동 데이터 확인

**5.** 첫 타겟 크리에이터: 수코(sookoh, @sookohaseyo) — 구독자 20.5만, 오사카·홍콩·대만 여행 콘텐츠 다수

**6.** 크리에이터 제안 앵글: "이미 만들어봤어요" + 정식 오픈 전 클로즈드 테스트 초대

**7.** 마이스토어 페이지 템플릿화 완료 — CREATOR 설정 블록만 수정하면 크리에이터별 페이지 생성 가능

**8.** TripSignal 바로가기 버튼 제거 — 모든 유입을 MRT 상품으로 집중

## 1. 배경 및 문제 정의

마이스토어는 여행 크리에이터 전용 바이오링크 서비스로, 크리에이터가 본인 여행 콘텐츠를 기반으로 MRT 상품을 큐레이션하고 팔로워의 실제 예약으로 어필리에이트 수익을 창출하는 구조임.

핵심 가설: 여행 크리에이터가 이 바이오링크 페이지를 활용해 어필리에이트 수익을 만들 수 있는가.

가설 검증 전 API 연동·실시간 가격 조회·자동 링크 생성 등 개발 범위가 과도하게 확대되어, 정작 "사람들이 클릭하고 구매하는가"라는 핵심 질문에 답하지 못하는 상황이 발생함.

---

## 2. 전략 방향 전환

### 2-1. 바이오 링크보다 자동DM이 효과적인 이유

바이오 링크 방식은 팔로워가 능동적으로 프로필에 방문해야 하는 수동적 구조임. 반면 자동DM은 팔로워가 댓글·DM으로 관심을 표현한 순간 링크를 전달하므로, 이미 구매 의도가 있는 사람에게만 도달하는 구조임. DM 오픈율은 바이오 링크 클릭률 대비 현저히 높음.

자동DM 툴(ManyChat 등)은 크리에이터가 직접 선택·운영하도록 위임함. 마이스토어 제공 범위는 링크(페이지)에 한정함.

### 2-2. 크리에이터가 DM으로 보낼 링크 선택지

코스 페이지보다 메인 페이지(쿠폰팩 노출)의 구매 전환율이 높다는 판단하에, 선택지를 2개로 정의함.

| 옵션 | 내용 | 특징 |
|------|------|------|
| A | 마이스토어 전체 페이지 | 쿠폰팩 바로 노출, 전환 유도 강함 |
| B | 특정 상품 링크 바로 | 맥락 맞는 직접 연결, 쿠폰 미노출 |

### 2-3. API 개발 포기 배경

MRT Open API(항공권 조회·숙소 조회·TNA 조회)와 MyLink API 구조를 파악했으나, 구현에 필요한 요소(크리에이터별 partner_id 관리·mylink_id 발급·서버사이드 인증 처리)가 MVP 가설 검증 범위를 초과함. 가설이 검증된 이후 단계에서 연동하는 것이 적절함.

## 3. 크리에이터 섭외 전략

### 3-1. 인스타그래머

- 수경님이 직접 DM 탭핑
- 수경님 개인 인스타에 서비스 소개 게시 → 지인 크리에이터 자연 유입

### 3-2. 유튜버

- 여행 루트 콘텐츠 다수 보유 채널 타겟
- 이메일 콜드아웃리치

### 3-3. 제안 메시지 핵심 앵글

- 광고 제안이 아닌 서비스 테스트 초대 형식
- 정식 오픈 전 소수 크리에이터 클로즈드 테스트
- "이미 만들어봤어요" + 목업 링크 첨부
- 전용 쿠폰 발급 + 어필리에이트 수익 공유
