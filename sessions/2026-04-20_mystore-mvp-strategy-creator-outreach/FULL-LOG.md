## Phase 2: 자동DM 방식 채택 + 배포 채널 전략 수립

수경님이 바이오 링크보다 자동DM 방식이 더 효과적이라는 판단을 제시함.

- 바이오 링크: 팔로워가 프로필을 능동적으로 방문해야 하는 수동 구조
- 자동DM: 댓글/DM으로 관심을 표현한 순간 링크 전달 → 이미 구매 의도가 있는 사람에게만 도달

DM 툴(ManyChat 등) 선택과 운영은 크리에이터 자율에 맡기기로 결정. 마이스토어가 제공하는 범위는 링크(페이지)에 한정함.

크리에이터가 DM으로 보낼 링크 선택지를 2개로 정의:
- 옵션A: 마이스토어 메인 페이지 (쿠폰팩 바로 노출, 전환 유도 강함)
- 옵션B: 특정 상품 링크 직접 연결 (맥락 맞는 직접 연결, 쿠폰 미노출)

코스 페이지 링크는 구매 전환 확률이 메인 페이지보다 낮아 지양하기로 함.

## Phase 3: 크리에이터 섭외 전략 + 메시지 초안 작성

섭외 채널 2개를 확정함:
1. 인스타그래머: 수경님이 직접 DM 탭핑 + 본인 인스타 서비스 소개 게시(지인 크리에이터 자연 유입)
2. 유튜버: 이메일 콜드아웃리치

제안 앵글: "이미 만들어봤어요" + 정식 오픈 전 클로즈드 테스트 초대 + 전용 쿠폰 + 어필리에이트 수익 공유.

인스타그래머 DM 초안(한국어, 캐주얼)과 유튜버 이메일 초안(프로페셔널, 비즈니스 톤)을 각각 작성함. 두 초안 모두 수경님 확인 완료.

## Phase 4: 어필리에이트 추적 방식 + API 범위 검토

어필리에이트 추적을 위한 기술 옵션을 검토함.

MRT MyLink API 구조 파악:
- mylink_id: 링크별 고유 ID (크리에이터 ID가 아님 — 이 부분 초기 혼동 있었음, 수경님이 정정)
- partner_id: 크리에이터/파트너 계정 고유 ID
- POST /v1/mylink + Bearer 인증 필요

MRT Open API(항공권/숙소/TNA) 연동도 검토했으나, 크리에이터별 partner_id 관리 + mylink_id 발급 + 서버사이드 인증 처리 등 개발 범위가 MVP 가설 검증을 초과한다는 판단 도출.

수경님: "우선 그렇게 되면 개발 스케일이 너무 커지지 않아? 우리의 목적에서 벗어나는 것 같은 느낌이 계속 드네"

최종 결정: API 연동 전면 포기. MVP 추적은 utm_content 파라미터 수동 부착으로 대체.

## Phase 5: TripSignal CORS 이슈 대응 + CTA 버튼 제거

배포된 Vercel 사이트에서 TripSignal API 호출 시 CORS 오류 발생 확인. 4개 도시 모두 실패.

Vercel 서버리스 프록시(`api/prices/[city].js`)를 생성하여 임시 해결. fetch URL을 외부 URL에서 내부 `/api/prices/${cityKey}` 경로로 변경.

수경님이 "괜히 편법 안써도 돼, 만약 안되면 태욱님께 여쭤볼게"로 의사를 전달함. 근본 해결은 TripSignal 개발자에게 CORS 헤더 추가 요청 예정.

"더 많은 최저가 경비 찾기 = TripSignal 바로가기 링크는 없애자. MRT 상품으로 모든 유입을 몰아야 하니까" → CTA 버튼 및 파인프린트 링크 제거 완료.
