## 주요 논의

### 1. 코스 섹션 정보 과밀 문제

- 사진 스트립 + 제목 + 메타 + 총경비 박스 + 항목 스와이프 + 버튼 2개 → 중복과 포화 느낌이 강했음.
- 1차 5안·2차 5안을 각각 반려한 뒤 수경님이 직접 작성한 `card-candidates.html`의 Option 02(선택 질문형)을 출발점으로 삼기로 함.
- "3박 4일" 등 기간 정보는 제목에서 분리해 뱃지로, 시즌(예: 26년 5월 여행)을 추가 뱃지로 표기하는 방향으로 합의함.

### 2. 캡션 방식 M2 채택

- V5(구분선+우측정렬) 단독 안 대신 V1(숫자 강조)을 합친 M2(좌 라벨 2줄 + 우 26px 숫자)로 확정함.
- 라벨은 "실시간 · 1인 기준" + "{도시명} 최저 총비용" 2줄 구성. 도시명 변수 삽입 가능 구조를 염두에 둠.

### 3. 국기 표기 & 말풍선 텍스트 블록

- 국가명 태그("🇯🇵 일본")보다 제목 옆 원형 뱃지가 깔끔하다는 판단으로 되돌림. 뱃지 크기는 34→46px 로 확대해 제목 컬럼 높이와 균형을 맞춤.
- 크리에이터 한마디 블록은 V3 말풍선(아바타 + 꼬리)로 확정. 배경은 다른 흰 카드와 구분하기 위해 `#FFF5E8` + 테두리 `#FFD9AE` 조합으로 지정함.

### 4. TripSignal 연동 구조

- 공개 엔드포인트 `/api/prices/osaka` 발견 후 CORS 허용 상태 확인. 1인 기준 값은 항공 그대로 + 숙소(per-night) × 박 수로 산출 가능함을 합의함.
- 응답 구조 방어(`!res.ok`, `json.data.flights/hotels` 가드, AbortController 8초)를 추가함.

### 5. 경비 패널 재편

- 식비를 맨 위로 올리자는 1차 제안은 반려됨. 숙소 아래(항공→숙소→식비) 배치로 확정.
- 교통 항목은 투어티켓 그룹 내 "오사카 지하철 2일권" 등과 중복되어 삭제함.
- 투어티켓 그룹 헤더는 "🎫 투어 티켓"만 남기고 "셀렉트유가 직접 구매 · 4개" 서브카피는 제거함.

### 6. 필수템 OG 자동화

- microlink `embed=image.url`은 heic도 jpg로 자동 변환되어 링크 탭에서 검증됨.
- 필수템 상품 3개 URL 실테스트에서 전부 jpg로 반환됨을 확인함.
- `loadProductOg()` 별도 함수로 분리, 이모지 fallback 유지 형태로 구현함.

### 7. PC 캐러셀 화살표

- 최초 오버레이 방식은 히어로 이미지 위에 떠서 시각적으로 이상했음.
- 도트 좌우 페이지네이션 스타일(`‹ • • • • ›`)로 재배치하고 모바일에서는 숨김 처리함.
