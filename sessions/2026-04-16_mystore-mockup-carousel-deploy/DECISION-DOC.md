# 마이스토어 목업 — 여행 카드 캐러셀·실시간 가격·OG 자동화 배포

- 날짜: 2026-04-16
- 주제: 메인 목업(index.html)의 코스 섹션 구조 개편 및 Vercel 배포
- 유형: 기획/디자인/구현 결정
- 배경: 기존 단일 코스 카드(오사카&교토)에 정보가 과밀하고, 다른 여행지 확장이 불가능한 구조였음. 또한 항공·숙소 경비가 하드코딩이고, 필수템·링크 카드의 비주얼이 이모지에 머물러 상품 구매 유도가 약했음.

## 최종 결정 사항

1. **여행 카드 캐러셀(M2) 채택** — 카드 1장을 4개 여행지 스와이프 캐러셀로 전환함. 카드 구조는 국기 원형 뱃지 + 여행지 제목 + 기간/시즌 뱃지 + 2버튼(일정표/상세 경비) + 우측정렬 캡션으로 확정함.
2. **TripSignal 공개 API 연동** — `/api/prices/{city}?directOnly=false` 를 클라이언트 fetch로 호출해 오사카 실시간 최저가(항공 1인 왕복 + 숙소 1박×3)를 메인 카드 총경비와 경비 패널에 반영함.
3. **OG 이미지 자동 로딩 범위 확장** — microlink `embed=image.url` 로 heic→jpg 변환 방식 적용. 링크 탭 OG 카드 외 필수템 탭 상품 카드까지 확장함. 로드 실패 시 이모지/플레이스홀더 fallback 유지.
4. **경비 패널 구조 재편** — 항공 → 숙소 → 식비 순으로 배치함. 교통 항목은 제거하고 투어티켓 그룹(이심·하루카스·라피트·지하철 2일권) 하위에 상품별 예약 버튼을 개별 노출함.
5. **PC 캐러셀 네비게이션 추가** — 도트 좌우에 페이지네이션 스타일 화살표를 배치함. 모바일에서는 `@media (hover: none)` 으로 숨기고 스와이프만 사용함.
6. **프로덕션 배포 유지** — `mystore-mockup.vercel.app` alias에 당일 3건 배포함(카드 캐러셀 반영, 화살표 페이지네이션 수정, 필수템 OG 자동화).

## 1. 배경 및 문제 정의

- 기존 코스 카드는 사진 스트립 + 제목 + 총경비 박스 + 2타일로 구성됨. 사용자 피드백은 "정보 포화·중복"이었음.
- 참고 레퍼런스는 인포크링크, Catno, RUMBO 등이었으나 테마 전면 교체 시도는 전부 반려됨. 결국 기존 톤(크림 + 오렌지 액센트)을 유지하고 구조만 개편하는 방향으로 수렴함.
- TripSignal(태욱님 작성)이 공개 배포된 상태였고 `/api/prices/{city}` 엔드포인트가 CORS 허용 상태였기에 실시간 가격 연동이 기술적으로 즉시 가능했음.
- 필수템 탭이 이모지 + 배경색 조합이라 상품의 실체감이 부족함. 링크 탭은 이미 microlink 기반 OG 로딩이 동작 중이었음.

## 3. 기술 구현 결정

**3.1 TripSignal 데이터 모델**

| 필드 | 의미 | 집계 방식 |
|------|-----|-----------|
| `flights[date][duration].price` | 1인 왕복 항공 | 그대로 사용 |
| `hotels[date][duration].price` | 1박당 객실료 | duration(=박) 만큼 곱셈 |
| 메타 `dateRange` | 향후 6개월 | 클라이언트 필터로 `now ~ +6M` 제한 |

- 1인 기준 디스플레이 채택. 2인 기준 전환은 차후 옵션으로 확장 가능.
- fetch에 `AbortController` 8초 타임아웃 + `!res.ok` 체크 + `json.data.flights/hotels` 가드 추가함.

**3.2 OG 이미지 로딩**

| 컴포넌트 | 방식 |
|----------|-----|
| 링크 탭 `.ls-og` | microlink JSON fetch로 title + 이미지 URL을 embed 프록시에 파이프함 |
| 필수템 탭 `.pc-img` | 별도 함수 `loadProductOg()` 분리. 이미지 onload 시 backgroundImage 설정, 이모지 span 숨김. 실패 시 이모지 fallback 유지 |

- microlink 무료 티어 50req/day 제한 인지함. 실제 사용 페이지 뷰에서 재 fetch가 반복될 경우 캐시(localStorage 등) 도입 검토 필요.

**3.3 캐러셀 UX**

| 대상 | 구현 |
|------|-----|
| 모바일(터치) | `scroll-snap-type: x mandatory`, 도트 인디케이터, 화살표 숨김 |
| PC(hover) | 도트 좌우에 28px 원형 페이지네이션 화살표, 끝단에서 opacity 0.3로 비활성화 |
| 스크롤 추적 | `scroll` 이벤트에서 cardWidth 기반 index 계산 |
