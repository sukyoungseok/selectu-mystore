- **날짜:** 2026-04-13
- **참석:** 마케팅파트너팀
- **주제:** 마이스토어 어필리에이트 연동을 위한 MyLink API 구조 파악

---

## 안건

마이스토어 MVP에서 크리에이터 어필리에이트 추적 인프라로 기존 MyLink API 재활용 가능 여부 및 구체적 구조 확인.

---

## 주요 논의

### 1. MyLink API 구조 파악

공식 문서(docs.myrealtrip.com) JS 번들 분석으로 실제 구조 확인.

mylink_id가 파트너 공통 ID가 아닌 링크별 고유 ID임이 확인됨. 어필리에이트 링크 생성 시마다 POST /v1/mylink 호출이 필요하며, Authorization: Bearer {API 키} 인증 방식을 사용함.

### 2. 연동 방식 논의

크리에이터 각자 파트너 계정을 보유하고 해당 API 키로 링크를 생성하는 방안 A와, 마이스토어 서비스 계정 하나가 모든 링크를 생성 후 내부 정산하는 방안 B를 검토. 수익이 크리에이터 계정에 직접 귀속되는 구조가 자연스럽고 기존 마파 인프라와 일치하여 방안 A 확정.

### 3. MVP 온보딩 플로우 영향

크리에이터 온보딩 시 파트너 계정 연동 단계가 추가됨. 화이트글러브 방식이면 운영팀이 대신 처리 가능하여 MVP 공수 절감 가능. 셀프 편집 툴 방식이면 파트너 계정 연동 UI 별도 구현 필요.
